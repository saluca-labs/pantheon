# Phase 16 Plan 02 — CoT Storage & Proof APIs

## Requirements
- **ALETH-10**: CoT content stored encrypted (AES-256-GCM, tenant DEK), opt-in per tenant
- **ALETH-11**: Prove-without-exposing: API returns hash proof without content for compliance audits
- **ALETH-12**: Chain verification API: validate integrity over entry range, return first broken link
- **ALETH-16**: Proof export: generate hashed audit proof document (JSON) for compliance

**Encryption pattern**: The codebase had an envelope encryption module at `src/tiresias/encryption/` (AES-256-GCM with DEK/KEK) but only `.pyc` files remain. For this plan, implement a self-contained AES-256-GCM encryption module within the aletheia package. Use the `cryptography` library (already in requirements.txt as a transitive dependency of other packages). The encryption approach:
- Each tenant has a Data Encryption Key (DEK) derived from a master secret + tenant_id via HKDF
- Master secret comes from env var `ALETHEIA_MASTER_KEY` (32 bytes, base64-encoded)
- DEK = HKDF-SHA256(master_key, salt=tenant_id_bytes, info=b"aletheia-cot-dek", length=32)
- Encrypt: AES-256-GCM(DEK, 12-byte random nonce, plaintext) -> (ciphertext, nonce, tag)

**API patterns** (from `src/siem/router.py`):
- FastAPI APIRouter with prefix and tags
- Pydantic BaseModel for request/response schemas
- HTTPException for errors
- Standard CRUD-style endpoints

**SoulWatch event pattern**: Events are ingested via the audit table polling (`soulWatch/src/pipeline/ingestion.py`) or direct POST. New event types just need to appear in the audit trail.

## Implementation Steps

### Step 1: Create Aletheia encryption module

**File: `src/aletheia/encryption.py`**

Implement tenant-scoped AES-256-GCM encryption:

```python
import os
import base64
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes

def get_master_key() -> bytes:
    """Load master key from ALETHEIA_MASTER_KEY env var (base64-encoded 32 bytes)."""
    key_b64 = os.environ.get("ALETHEIA_MASTER_KEY")
    if not key_b64:
        raise RuntimeError("ALETHEIA_MASTER_KEY not set")
    key = base64.b64decode(key_b64)
    if len(key) != 32:
        raise RuntimeError("ALETHEIA_MASTER_KEY must be 32 bytes")
    return key

def derive_tenant_dek(master_key: bytes, tenant_id: str) -> bytes:
    """Derive a per-tenant DEK using HKDF-SHA256."""
    hkdf = HKDF(
        algorithm=hashes.SHA256(),
        length=32,
        salt=tenant_id.encode("utf-8"),
        info=b"aletheia-cot-dek",
    )
    return hkdf.derive(master_key)

def encrypt_content(dek: bytes, plaintext: bytes) -> tuple[bytes, bytes, bytes]:
    """AES-256-GCM encrypt. Returns (ciphertext, nonce, tag)."""
    nonce = os.urandom(12)
    aesgcm = AESGCM(dek)
    # AESGCM.encrypt returns ciphertext+tag concatenated (tag is last 16 bytes)
    ct_with_tag = aesgcm.encrypt(nonce, plaintext, None)
    ciphertext = ct_with_tag[:-16]
    tag = ct_with_tag[-16:]
    return ciphertext, nonce, tag

def decrypt_content(dek: bytes, ciphertext: bytes, nonce: bytes, tag: bytes) -> bytes:
    """AES-256-GCM decrypt. Returns plaintext."""
    aesgcm = AESGCM(dek)
    ct_with_tag = ciphertext + tag
    return aesgcm.decrypt(nonce, ct_with_tag, None)
```

### Step 2: Create CoT content storage service

**File: `src/aletheia/storage.py`**

Service layer that handles opt-in encrypted content storage:

1. **`class CotContentStorage`**:
   - `__init__(self, session_factory)`
   - `async store_content(self, chain_entry_id: UUID, tenant_id: UUID, reasoning_text: str) -> bool`:
     a. Derive tenant DEK: `derive_tenant_dek(get_master_key(), str(tenant_id))`
     b. Encrypt: `encrypt_content(dek, reasoning_text.encode("utf-8"))`
     c. INSERT `AletheiaCoTContent` row with `encrypted_content`, `content_nonce`, `content_tag`
     d. UPDATE the `AletheiaCoTChain` row: set `content_stored=True`, `content_ref=str(content.id)`
     e. Return True on success
   - `async retrieve_content(self, chain_entry_id: UUID, tenant_id: UUID) -> str | None`:
     a. SELECT `AletheiaCoTContent` by `chain_entry_id`
     b. If not found, return None
     c. Derive tenant DEK
     d. Decrypt: `decrypt_content(dek, row.encrypted_content, row.content_nonce, row.content_tag)`
     e. Return decoded plaintext

2. **Tenant opt-in check**: The storage service checks a tenant config flag before storing content. Add a helper `async is_content_storage_enabled(session, tenant_id) -> bool` that checks the tenant metadata JSON field for `{"aletheia": {"store_cot_content": true}}`. Default is False (hash-only mode).

### Step 4: Create Aletheia CoT API router

**File: `src/aletheia/router.py`**

FastAPI router with prefix `/v1/aletheia/cot`, tags `["Aletheia CoT"]`:

**Pydantic schemas** (define in same file or `src/aletheia/schemas.py`):

```python
class ChainEntryResponse(BaseModel):
    id: str
    chain_id: str
    entry_index: int
    request_id: str
    timestamp: str
    model: str
    provider: str
    agent_id: str | None
    cot_hash: str
    cot_token_count: int
    cot_byte_count: int
    prev_hash: str
    entry_hash: str
    content_stored: bool

class ChainVerifyResponse(BaseModel):
    valid: bool
    entries_checked: int
    first_broken_index: int | None
    error: str | None

class ProofDocument(BaseModel):
    version: str  # "1.0"
    generated_at: str
    tenant_id: str
    chain_id: str
    entries: list[dict]  # Each entry: {entry_index, request_id, timestamp, model, cot_hash, entry_hash, prev_hash, cot_token_count}
    chain_valid: bool
    verification_hash: str  # SHA-512 of the entire proof payload for tamper detection

class ProofExportRequest(BaseModel):
    request_ids: list[str] | None = None
    from_index: int | None = None
    to_index: int | None = None
    include_content: bool = False  # If True and content stored, include decrypted content
```

**Endpoints:**

1. **`GET /v1/aletheia/cot/chain`** -- List chain entries (ALETH-11 partial)
   - Query params: `tenant_id`, `from_ts`, `to_ts`, `agent_id`, `model`, `limit=50`, `offset=0`
   - Returns: `{"entries": [ChainEntryResponse], "total": int}`
   - This is the "prove-without-exposing" default: returns hashes, never content

2. **`GET /v1/aletheia/cot/chain/{request_id}`** -- Single entry by request_id
   - Returns: ChainEntryResponse (hash proof only)

3. **`GET /v1/aletheia/cot/chain/{request_id}/content`** -- Decrypted content (ALETH-10)
   - Requires `audit:read` scope (check via auth context)
   - If content not stored: 404
   - If stored: decrypt and return `{"request_id": str, "reasoning_content": str, "token_count": int}`
   - Log content access in audit trail

4. **`GET /v1/aletheia/cot/chain/verify`** -- Chain verification (ALETH-12)
   - Query params: `tenant_id`, `from_index`, `to_index`
   - Calls `verify_chain_range()` from chain.py
   - Returns: ChainVerifyResponse

5. **`POST /v1/aletheia/cot/proof`** -- Proof export (ALETH-16)
   - Body: ProofExportRequest
   - Generates a JSON proof document containing:
     - All chain entries in the requested range (hashes only unless `include_content=True`)
     - Chain verification result
     - A `verification_hash`: SHA-512 of the serialized entries array, so the recipient can verify document integrity
   - Returns: ProofDocument
   - If `include_content=True`: decrypt and include content for entries that have it stored (requires `audit:read` scope)
