## Implementation Steps

### Step 1: Create ToolPolicy schema and YAML loader
**File:** `src/aletheia/tool_policy.py`

Following the exact pattern from `src/aletheia/cot_policy.py`:

1. Define dataclasses:
   - `ToolPolicyRule`: name (str), match (MatchSpec), action (str: "allow"|"deny"|"warn"), reason (str), rate_limit (optional RateLimitSpec)
   - `MatchSpec`: commands (list[str] -- supports glob via fnmatch), args_pattern (optional regex str, compiled at load time)
   - `RateLimitSpec`: max_per_minute (int), max_per_hour (int)
   - `AgentOverride`: override_rules (list of dicts with name + action), default_action (optional str), allowed_commands (optional list[str])
   - `ToolPolicy`: name, default_action ("allow"), rules (list[ToolPolicyRule]), agent_overrides (dict[str, AgentOverride]), enabled (bool)

2. `load_tool_policy(path: str) -> Optional[ToolPolicy]`:
   - Validate `apiVersion: tiresias/v1`, `kind: ToolPolicy`
   - Parse metadata.name, spec.default_action, spec.rules, spec.agent_overrides
   - Compile args_pattern regex at load time (store as `re.Pattern | None`)
   - Return None on invalid files (with structlog warning)

3. `load_tool_policies_from_dir(dir_path: str = "policies/tool") -> list[ToolPolicy]`:
   - Glob `*.yaml`, sort, load each, return enabled ones

### Step 2: Create ToolPolicyEngine evaluator
**File:** `src/aletheia/tool_policy_engine.py`

Following the pattern from `src/aletheia/cot_enforcer.py`:

1. Define `ToolPolicyResult` dataclass:
   - verdict: str ("allow"|"deny"|"warn")
   - rule_matched: Optional[str]
   - reason: str
   - override_applied: bool
   - rate_limited: bool

2. `ToolPolicyEngine` class:
   - `__init__(self, policies: list[ToolPolicy])` -- stores policies, initializes rate limit state (dict keyed by agent_id:rule_name -> deque of timestamps)
   - `evaluate(self, agent_id: str, tenant_id: str, command: str, args: list[str]) -> ToolPolicyResult`:
     a. For each policy, check agent_overrides first:
        - If agent has override with `default_action: deny` and `allowed_commands`, only allow listed commands (fnmatch)
        - If agent has `override_rules`, build a map of rule_name -> overridden_action
     b. Match command against each rule's `match.commands` using fnmatch. The full_command for matching is `command` (just the binary name) AND `command + " " + " ".join(args)` (for compound patterns like "gws admin users.delete")
     c. If rule has args_pattern, match against space-joined args string
     d. If rule has rate_limit, check sliding window counters (per agent_id + rule_name). If exceeded, escalate action to "deny" with reason "rate limit exceeded"
     e. Collect all matching rules. Deny wins over warn, warn wins over allow.
     f. If no rules match, use policy default_action
   - `_check_rate_limit(self, key: str, limit: RateLimitSpec) -> bool`:
     - Use `collections.deque` per key storing timestamps
     - Prune entries older than 1 hour
     - Check count in last minute and last hour against limits
     - Return True if limit exceeded

3. Module-level singleton functions (same pattern as cot_enforcer.py):
   - `_active_engine: Optional[ToolPolicyEngine] = None`
   - `init_tool_policy_engine(policy_dir: str = "policies/tool") -> None`
   - `get_active_engine() -> Optional[ToolPolicyEngine]`
   - `reload_tool_policies(policy_dir: str = "policies/tool") -> None` -- calls init_tool_policy_engine (thread-safe via GIL assignment)

### Step 3: Create default tool policy YAML
**File:** `policies/tool/default-enterprise.yaml`

Schema follows `policies/cot/require-reasoning.yaml` structure:
```yaml
apiVersion: tiresias/v1
kind: ToolPolicy
metadata:
  name: default-enterprise
  tier: enterprise
spec:
  default_action: allow
  rules:
    - name: block-destructive
      match:
        commands: ["rm", "gws admin users.delete", "kubectl delete", "git push --force", "mkfs", "dd"]
      action: deny
      reason: "Destructive operation requires explicit approval"
    - name: warn-data-access
      match:
        commands: ["gws drive files.export", "gws gmail users.messages.get"]
        args_pattern: ".*--(format|type)=(raw|full).*"
      action: warn
      reason: "Full content export -- verify data classification"
    - name: rate-limit-api
      match:
        commands: ["gws *", "gh api *"]
      rate_limit:
        max_per_minute: 60
        max_per_hour: 500
      action: allow
  agent_overrides:
    alfred-main:
      override_rules:
        - name: block-destructive
          action: warn
    untrusted-agent:
      default_action: deny
      allowed_commands:
        - "gws gmail users.messages.list"
        - "gws drive files.list"
        - "gws calendar events.list"
```

### Step 4: Create POST /v1/aletheia/tool/evaluate endpoint
**File:** `src/aletheia/tool_evaluate_router.py`

1. FastAPI APIRouter with prefix `/v1/aletheia/tool`, tags `["Aletheia Tool Policy"]`
2. Pydantic request model `ToolEvaluateRequest`:
   - agent_id: str
   - tenant_id: str
   - command: str
   - args: list[str]
   - context: Optional[dict] = None (working_directory, session_id)
3. Pydantic response model `ToolEvaluateResponse`:
   - verdict: str
   - rule_matched: Optional[str] = None
   - reason: str
   - override_available: bool = False
   - logged: bool = True
4. `POST /evaluate` endpoint:
   - Import and call `get_active_engine()` from tool_policy_engine
   - If no engine, return `{"verdict": "allow", "reason": "no policy engine active", "logged": False}`
   - Call `engine.evaluate(request.agent_id, request.tenant_id, request.command, request.args)`
   - Return ToolEvaluateResponse mapped from result
   - Use structlog to log every evaluation (agent_id, command, verdict, rule_matched)
5. `POST /reload` endpoint (admin only):
   - Calls `reload_tool_policies()`
   - Returns count of loaded policies
