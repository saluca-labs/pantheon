```python
class CrossTenantCoTEntry(BaseModel):
    id: str
    chain_id: str
    entry_index: int
    request_id: str
    tenant_id: str
    tenant_name: Optional[str] = None
    timestamp: str
    model: str
    provider: str
    agent_id: Optional[str] = None
    cot_hash: str
    cot_token_count: int
    entry_hash: str
    content_stored: bool

class CrossTenantCoTResponse(BaseModel):
    entries: list[CrossTenantCoTEntry]
    total: int
    tenant_count: int

class PolicyPushRequest(BaseModel):
    target_tenant_ids: list[str]  # UUIDs of child tenants to push to
    policy_yaml: str  # YAML content to push

class PolicyPushResult(BaseModel):
    tenant_id: str
    tenant_name: Optional[str] = None
    status: str  # "success" | "error"
    detail: Optional[str] = None

class PolicyPushResponse(BaseModel):
    results: list[PolicyPushResult]
    success_count: int
    error_count: int
```

**In src/mssp/router.py**, add two new endpoints following the exact pattern of cross_tenant_matches:

**Endpoint 1: GET /v1/mssp/aletheia/cot**
Cross-tenant CoT chain query. Implementation:
1. Extract caller tenant via _get_caller_tenant_id(request)
2. Get db session, verify mssp tier via _require_mssp_tier(db, caller_id)
3. Get child IDs via get_child_tenant_ids(db, caller_id)
4. If no children, return empty CrossTenantCoTResponse
5. Query AletheiaCoTChain where tenant_id.in_(child_ids), order by timestamp desc, with limit (default 50, max 200) and offset query params
6. For each entry, join or subquery to get tenant_name from SoulTenant
7. Count total matching entries for pagination
8. Return CrossTenantCoTResponse with entries, total, tenant_count (distinct tenant_ids in results)

Import AletheiaCoTChain from src.aletheia.models. Use the same SQLAlchemy async pattern as existing endpoints (select, where, order_by, limit, offset).

**Endpoint 2: POST /v1/mssp/aletheia/policies/push**
Managed policy push to child tenants. Implementation:
1. Extract caller tenant, verify mssp tier (same pattern)
2. Get child IDs
3. Validate that all target_tenant_ids are in child_ids (raise 403 if any target is not a child)
4. For each target tenant: store the policy YAML in the tenant metadata_ JSON column under key "aletheia_tool_policy_yaml". This is the same metadata_ pattern used for branding in Phase 14 (SoulTenant.metadata_ is a JSON column).
5. Return PolicyPushResponse with per-tenant results

The policy push stores YAML in metadata_ for now. The tiresias-exec agent on each child tenant reads policy from this field on next reload. This is a pragmatic first implementation -- a dedicated policy table can be added later.

Note: Do NOT modify the ROUTE_FEATURES dict in feature_gate.py. The /v1/mssp prefix is already mapped to "tenant_hierarchy" (mssp+), so these new endpoints under /v1/mssp/aletheia/* are automatically gated.

Add appropriate structlog calls: logger.info("mssp.aletheia.cot_query", ...) and logger.info("mssp.aletheia.policy_push", ...).
