# Phase 16 Plan 01 — CoT Extraction & Hash Chain Core

## Requirements
- **ALETH-08**: CoT intercept extracts reasoning traces from Anthropic thinking blocks, OpenAI reasoning tokens, Gemini thinking parts
- **ALETH-09**: CoT hash chain: SHA-512 linked entries, tamper-evident, per-tenant chain with genesis anchor

## Context

**SoulGate proxy pipeline** (`soulGate/src/proxy/gateway.py`):
- `process_request()` handles the full proxy lifecycle: auth -> access -> rate limit -> circuit breaker -> inspect -> forward -> audit
- Response body is available as `upstream_response.content` after the httpx call (step 6)
- CoT intercept inserts in the **response path**, after receiving the upstream response and before returning to the agent
- The response passes through unmodified -- CoT intercept is read-only on the response

**Database models** follow SQLAlchemy 2.0 mapped_column style (see `soulGate/src/database/models.py`, `src/database/models.py`):
- UUID primary keys via `mapped_column(Uuid, primary_key=True, default=_uuid_default)`
- Timezone-aware datetimes, JSON columns, index definitions in `__table_args__`
- Tables prefixed by subsystem (`_soulgate_*`, `_soulauth_*`)

**Audit logging** (`soulGate/src/audit/logger.py`):
- Queue-based async batch insert pattern
- `enqueue_log_entry()` adds to asyncio.Queue, background `_flush_loop()` writes in batches

**Spec reference**: Section 3.2.2 (provider extraction), 3.2.3 (hash chain structure), 3.2.4 (storage model), 3.2.5 (SoulGate integration)

### Step 2: Create provider-specific CoT extractors

**File: `src/aletheia/extractors.py`**

Define a dataclass for extraction results:

```python
@dataclass
class CotExtraction:
    provider: str          # "anthropic" | "openai" | "gemini" | "local"
    reasoning_text: str | None  # Full reasoning content (None for OpenAI)
    token_count: int       # Reasoning token count
    byte_count: int        # Byte length of reasoning text (0 if unavailable)
    model: str             # Model identifier from response
    has_reasoning: bool    # Whether reasoning was detected
```

Implement four extractors plus a dispatcher:

1. **`extract_anthropic(body: dict) -> CotExtraction | None`**
   - Look for `content` array entries where `type == "thinking"`
   - Concatenate all `.thinking` values
   - Token count: `usage.cache_creation_input_tokens` or estimate `len(text) // 4`
   - Model: `body.get("model", "unknown")`

2. **`extract_openai(body: dict) -> CotExtraction | None`**
   - Check `usage.completion_tokens_details.reasoning_tokens`
   - If > 0: reasoning occurred, content not available from API
   - Set `reasoning_text=None`, `has_reasoning=True`, `token_count=reasoning_tokens`

3. **`extract_gemini(body: dict) -> CotExtraction | None`**
   - Check `candidates[0].content.parts` for `thought == True`
   - Concatenate all thought-part `.text` values

4. **`extract_local(body: dict) -> CotExtraction | None`**
   - Regex: `re.findall(r"<thinking>(.*?)</thinking>", text, re.DOTALL)`
   - Best-effort only

5. **`extract_cot(provider: str, body: dict) -> CotExtraction | None`**
   - Dispatch to correct extractor based on provider
   - Wrap in try/except, return None on any error, log warning

6. **`detect_provider(upstream_name: str) -> str`**
   - Normalize upstream name to one of: "anthropic", "openai", "gemini", "local"
   - Match on substrings: "anthropic"/"claude" -> "anthropic", "openai"/"gpt"/"o1"/"o3" -> "openai", "gemini"/"google" -> "gemini"
   - Default: "local"

### Step 3: Create hash chain engine

**File: `src/aletheia/chain.py`**

1. **Constants:**
   - `GENESIS_PREFIX = "genesis::"`
   - Use `hashlib.sha512` throughout

2. **Pure functions (no I/O, easily testable):**
   - `compute_cot_hash(reasoning_text: str | None, token_count: int) -> str` -- SHA-512 of content, or SHA-512 of `"reasoning_tokens::{count}"` for OpenAI case
   - `compute_entry_hash(entry_index: int, request_id: str, timestamp: str, cot_hash: str, prev_hash: str) -> str` -- SHA-512 of `"{entry_index}||{request_id}||{timestamp}||{cot_hash}||{prev_hash}"`
   - `compute_genesis_hash(tenant_id: str) -> str` -- SHA-512 of tenant_id string

3. **`class CotChainWriter`** -- manages appending to a tenant's chain:
   - `__init__(self, session_factory, tenant_id: uuid.UUID)`
   - `async get_or_create_chain(self) -> uuid.UUID` -- query for existing chain_id for this tenant, or create genesis entry and return new chain_id
   - Genesis entry: `entry_index=0`, `cot_hash=SHA-512("genesis")`, `prev_hash=compute_genesis_hash(tenant_id)`, model="system", provider="system"
   - `async append(self, extraction: CotExtraction, request_id: uuid.UUID, agent_id: str | None) -> AletheiaCoTChain` -- atomically:
     a. Lock: `SELECT ... FROM aletheia_cot_chain WHERE tenant_id=? AND chain_id=? ORDER BY entry_index DESC LIMIT 1 FOR UPDATE`
     b. `new_index = last.entry_index + 1`
     c. `prev_hash = last.entry_hash`
     d. `cot_hash = compute_cot_hash(extraction.reasoning_text, extraction.token_count)`
     e. `entry_hash = compute_entry_hash(new_index, str(request_id), timestamp_iso, cot_hash, prev_hash)`
     f. INSERT new row, commit, return

4. **`async verify_chain_range(session, tenant_id, chain_id, from_index, to_index) -> dict`**
   - Fetch entries in [from_index, to_index] ordered by entry_index ASC
   - For each entry: recompute entry_hash, verify matches stored value
   - For consecutive entries: verify entry[i].prev_hash == entry[i-1].entry_hash
   - Return `{"valid": bool, "entries_checked": int, "first_broken_index": int | None, "error": str | None}`

### Step 4: Integrate CoT intercept into SoulGate gateway

**File: `soulGate/src/proxy/gateway.py`** -- modify `process_request()`

Insert CoT intercept after receiving the upstream response (after step 6 forward, before building the return Response). This is the section where `upstream_response` is available:

```python
# === CoT Intercept (read-only, fire-and-forget) ===
if response_body and response_status and 200 <= response_status < 300:
    try:
        import json as _json
        body_json = _json.loads(response_body)
        from src.aletheia.extractors import extract_cot, detect_provider
        provider = detect_provider(upstream.name)
        extraction = extract_cot(provider, body_json)
        if extraction and extraction.has_reasoning:
            from src.aletheia.chain import CotChainWriter
            writer = CotChainWriter(_get_session_factory(), auth_result.tenant_id)
            # Generate a request_id for this proxied call
            import uuid as _uuid
            req_id = _uuid.uuid4()
            asyncio.create_task(
                _safe_cot_append(writer, extraction, req_id, auth_result)
            )
    except Exception:
        logger.warning("cot_intercept.skipped", exc_info=True)
```

Add helper at module level:

```python
async def _safe_cot_append(writer, extraction, request_id, auth_result):
    """Fire-and-forget CoT chain append with error handling."""
    try:
        entry = await writer.append(extraction, request_id, auth_result.persona_id)
        logger.debug("cot_intercept.appended", entry_index=entry.entry_index, tenant_id=str(auth_result.tenant_id))
    except Exception as e:
        logger.error("cot_intercept.append_failed", error=str(e))
```

Also add `_get_session_factory()` that returns the database session factory (import from `soulGate.src.database.connection` or pass via app state).

**Critical constraints:**
- CoT intercept MUST NOT modify the response body
- CoT intercept MUST NOT delay the response (fire-and-forget via asyncio.create_task)
- CoT intercept MUST NOT raise exceptions that affect the proxy flow (wrapped in try/except)
