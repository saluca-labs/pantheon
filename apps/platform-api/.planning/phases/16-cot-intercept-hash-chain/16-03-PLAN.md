# Phase 16 Plan 03 — CoT Policy Enforcement

## Requirements
- **ALETH-17**: CoT policy enforcement: enterprise can mandate thinking enabled (inject/reject/warn modes), proxy-enforced, cost-tagged

## Context

**Depends on Plan 01** which creates:
- `src/aletheia/extractors.py` -- `detect_provider()` for identifying provider from upstream name
- CoT intercept wired into `soulGate/src/proxy/gateway.py` response path
- Aletheia feature gate entries in `src/middleware/feature_gate.py`

**SoulGate request pipeline** (`soulGate/src/proxy/gateway.py`):
- CoT policy enforcement operates on the **request path** (before forwarding to upstream), unlike CoT intercept which is on the response path
- The request body is available as `body_bytes` after step 5 (inspect payload)
- Policy enforcement must inject/modify the request body before step 6 (forward to upstream)
- Current pipeline steps: auth -> IP access -> rate limit -> circuit breaker -> inspect payload -> **[NEW: CoT policy]** -> forward -> **[EXISTING: CoT intercept]** -> audit -> return

**Policy loading pattern** (`src/policy/loader.py`, `src/policy/git_sync.py`):
- Policies are loaded from YAML files in the `policies/` directory
- The system supports hot-reload via git sync
- For CoT policies, follow the same pattern: YAML config in `policies/cot/` directory

**Spec reference**: Section 3.2.3 (CoT Policy Enforcement) -- three enforcement modes (inject/reject/warn), provider-specific injection fields, exemption patterns, cost tagging

## Implementation Steps

### Step 1: Define CotPolicy schema and loader

**File: `src/aletheia/cot_policy.py`**

Define the CoT policy data model and loader:

```python
@dataclass
class ProviderInjection:
    inject_field: str       # e.g., "thinking.enabled" for Anthropic
    inject_value: Any       # e.g., True
    budget_tokens: int      # max thinking budget to inject (e.g., 10000)

@dataclass
class ExemptionRule:
    model_pattern: str | None = None      # e.g., "haiku*"
    endpoint_pattern: str | None = None   # e.g., "/v1/embeddings"
    agent_pattern: str | None = None      # e.g., "linter-*"

@dataclass
class CotPolicy:
    name: str
    require_thinking: bool = True
    enforcement: str = "inject"  # "inject" | "reject" | "warn"
    providers: dict[str, ProviderInjection] = field(default_factory=dict)
    exemptions: list[ExemptionRule] = field(default_factory=list)
    enabled: bool = True
```

**Loader functions:**

1. **`load_cot_policy(policy_path: str) -> CotPolicy | None`**
   - Parse YAML file at `policies/cot/*.yaml`
   - Validate `apiVersion: tiresias/v1`, `kind: CotPolicy`
   - Return CotPolicy dataclass

2. **`load_cot_policies_from_dir(dir_path: str = "policies/cot") -> list[CotPolicy]`**
   - Load all `.yaml` files from directory
   - Return list of enabled policies

3. **`check_exemption(policy: CotPolicy, model: str, endpoint: str, agent_id: str | None) -> bool`**
   - Check if the request matches any exemption rule using `fnmatch` for glob patterns
   - Return True if exempt (policy should not apply)

### Step 2: Implement request body injection/rejection logic

**File: `src/aletheia/cot_enforcer.py`**

Core enforcement logic that operates on the raw request body:

1. **`class CotEnforcer`**:
   - `__init__(self, policies: list[CotPolicy])`
   - `evaluate(self, provider: str, body: dict, model: str, endpoint: str, agent_id: str | None) -> CotEnforcementResult`

2. **`@dataclass class CotEnforcementResult`**:
   - `action: str` -- "pass" (already has thinking), "inject" (thinking added), "reject" (blocked), "warn" (logged only), "exempt" (matched exemption)
   - `modified_body: dict | None` -- modified request body (only for "inject" action)
   - `policy_name: str | None`
   - `reason: str`
   - `cost_injected: bool` -- True if thinking was injected (for cost attribution)

3. **Evaluation logic in `evaluate()`**:
   a. For each enabled policy:
      - Check exemptions via `check_exemption()` -- if exempt, return action="exempt"
      - Check if request already has thinking enabled:
        - **Anthropic**: `body.get("thinking", {}).get("enabled")` is True
        - **OpenAI**: `body.get("reasoning_effort")` is set
        - **Gemini**: nested check for `body.get("generationConfig", {}).get("thinkingConfig", {}).get("thinkingBudget")` > 0
      - If already has thinking: return action="pass"
      - If not, apply enforcement mode:
        - **inject**: Deep-copy body, set provider-specific thinking fields, return action="inject" with modified_body
        - **reject**: Return action="reject" with reason="cot-policy-violation"
        - **warn**: Return action="warn" (request proceeds, violation logged)

4. **Provider-specific injection helpers**:
   - `_inject_anthropic(body: dict, injection: ProviderInjection) -> dict` -- set `body["thinking"] = {"type": "enabled", "budget_tokens": injection.budget_tokens}`
   - `_inject_openai(body: dict, injection: ProviderInjection) -> dict` -- set `body["reasoning_effort"] = injection.inject_value`
   - `_inject_gemini(body: dict, injection: ProviderInjection) -> dict` -- set nested `body["generationConfig"]["thinkingConfig"]["thinkingBudget"] = injection.budget_tokens`

### Step 3: Integrate enforcement into SoulGate gateway

**File: `soulGate/src/proxy/gateway.py`** -- modify `process_request()`

Insert CoT policy enforcement after payload inspection (step 5) and before upstream forwarding (step 6):

```python
# After step 5 (inspect payload), before step 6 (forward to upstream):

# === CoT Policy Enforcement ===
cot_policy_injected = False
if body_bytes and settings.cot_policy_enabled:
    try:
        import json as _json
        from src.aletheia.extractors import detect_provider
        from src.aletheia.cot_enforcer import CotEnforcer, get_active_enforcer

        enforcer = get_active_enforcer()
        if enforcer:
            provider = detect_provider(upstream.name)
            try:
                req_body = _json.loads(body_bytes)
            except (json.JSONDecodeError, UnicodeDecodeError):
                req_body = None

            if req_body:
                model = req_body.get("model", "")
                result = enforcer.evaluate(
                    provider=provider,
                    body=req_body,
                    model=model,
                    endpoint=path,
                    agent_id=auth_result.persona_id if auth_result else None,
                )

                if result.action == "reject":
                    blocked = True
                    block_reason = f"cot_policy: {result.reason}"
                    BLOCKS_TOTAL.labels(reason="cot_policy").inc()
                    return JSONResponse(
                        status_code=403,
                        content={
                            "detail": "CoT policy violation: thinking/reasoning must be enabled",
                            "policy": result.policy_name,
                            "blocked_by": "soulgate",
                        },
                        headers={"X-Tiresias-Reason": "cot-policy-violation"},
                    )

                if result.action == "inject" and result.modified_body:
                    body_bytes = _json.dumps(result.modified_body).encode("utf-8")
                    cot_policy_injected = True

                if result.action == "warn":
                    logger.warning(
                        "cot_policy.violation",
                        policy=result.policy_name,
                        agent_id=auth_result.persona_id if auth_result else None,
                        model=model,
                    )
    except Exception as e:
        logger.warning("cot_policy.evaluation_error", error=str(e))
```

Also update the audit log entry to include `cot_policy_injected` in the `threat_flags` or a new field, so cost attribution can identify policy-mandated thinking cost.

### Step 5: Add enforcer singleton with hot-reload

**File: `src/aletheia/cot_enforcer.py`** -- add module-level singleton:

```python
_active_enforcer: CotEnforcer | None = None

def init_cot_enforcer(policy_dir: str = "policies/cot") -> None:
    """Load policies and initialize the enforcer singleton."""
    global _active_enforcer
    policies = load_cot_policies_from_dir(policy_dir)
    if policies:
        _active_enforcer = CotEnforcer(policies)
        logger.info("cot_enforcer.initialized", policy_count=len(policies))
    else:
        _active_enforcer = None
        logger.info("cot_enforcer.no_policies")

def get_active_enforcer() -> CotEnforcer | None:
    return _active_enforcer

def reload_cot_policies(policy_dir: str = "policies/cot") -> None:
    """Hot-reload policies from disk."""
    init_cot_enforcer(policy_dir)
```

Wire initialization into SoulGate startup (or main.py lifespan):
```python
# In lifespan or startup:
if settings.cot_policy_enabled:
    from src.aletheia.cot_enforcer import init_cot_enforcer
    init_cot_enforcer(settings.cot_policy_dir)
```

### Step 6: Add cost tagging for injected thinking

When `cot_policy_injected=True`, the audit log entry should include a marker so cost attribution can separate policy-mandated reasoning cost:

In the `finally` block of `process_request()`, extend the `enqueue_log_entry()` call:
- If `threat_flags` dict exists, add `"cot_policy_injected": cot_policy_injected`
- If `threat_flags` is None and `cot_policy_injected` is True, set `threat_flags = {"cot_policy_injected": True}`

This allows downstream cost analysis to query `WHERE threat_flags->>'cot_policy_injected' = 'true'`.
