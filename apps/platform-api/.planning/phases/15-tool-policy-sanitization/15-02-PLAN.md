# Phase 15 Plan 02 — Response Sanitizer Engine

## Requirements
- **ALETH-06:** Response sanitizer scans CLI output for prompt injection. passthrough/warn/block modes; patterns extend PRH engine; < 50ms for outputs up to 1MB
- **ALETH-07:** Sanitizer patterns: indirect injection, credential exposure, encoding evasion. Multi-pass decoding (raw, base64, Unicode-normalized, HTML entities); tool-specific pattern categories

### What exists (Phase 16 -- pattern to follow)

### Spec requirements (section 3.3)
- Sanitizer must complete in < 50ms for outputs up to 1MB
- Multi-pass decode: raw -> base64 decoded -> Unicode normalized -> HTML entity decoded
- Modes: passthrough (no scan, still log), warn (scan + log, pass through), block (scan + replace output + store original encrypted)
- Blocked output: replace with `[BLOCKED: prompt injection detected in tool response]`
- Blocked output original: store encrypted in SoulWatch for forensics
- Pattern categories: direct injection (reuse PRH), indirect injection (new), credential exposure (new), encoding evasion (new), data exfiltration markers (adapted from PRH)

## Implementation Steps

### Step 1: Create sanitizer pattern definitions
**File:** `src/aletheia/sanitizer_patterns.py`

Extend PRH pattern structure for tool-output-specific threats. Use the same `PatternDef` TypedDict from `src/prh/patterns.py`.

1. Import `PatternDef` and `_compile` helper pattern (or redefine locally for independence).

2. Define new pattern categories for tool output scanning:

   **`tool_injection`** (direct injection in tool output -- adapted from PRH injection category):
   - Reuse top 5-6 highest-weight PRH injection patterns (ignore_previous, new_instructions, override_system, injection_marker, etc.)
   - Add tool-output-specific patterns:
     - `hidden_instruction`: `(?i)(you\s+must|you\s+should|please\s+now)\s+(ignore|disregard|forget|override)` (weight 0.85)
     - `embedded_system_prompt`: `(?i)<\|?(system|im_start|INST)\|?>.*?(ignore|override|disregard)` (weight 0.9)
     - `tool_output_directive`: `(?i)(IMPORTANT|NOTE|ATTENTION):\s*(ignore|disregard|override|forget)\s+(all\s+)?(previous|prior)` (weight 0.9)

   **`indirect_injection`** (hidden instructions in document content):
   - `zero_width_encoded`: pattern matching sequences of zero-width characters (U+200B, U+200C, U+200D, U+FEFF) that may encode instructions (weight 0.8)
   - `invisible_text_block`: consecutive invisible Unicode categories (Cf, Mn) exceeding 10 chars (weight 0.7)
   - `rtl_override_trick`: RTL override characters (U+202E, U+2066-U+2069) used to visually hide text (weight 0.75)
   - `homoglyph_substitution`: mixed script detection (Latin + Cyrillic in same word) (weight 0.6)

   **`credential_exposure`** (secrets/keys in tool output):
   - `aws_key`: `(?i)AKIA[0-9A-Z]{16}` (weight 0.95)
   - `github_token`: `(ghp|gho|ghu|ghs|ghr)_[A-Za-z0-9_]{36,}` (weight 0.95)
   - `generic_api_key`: `(?i)(api[_-]?key|apikey|secret[_-]?key)\s*[:=]\s*['"]?[A-Za-z0-9\-_]{20,}` (weight 0.85)
   - `jwt_token`: `eyJ[A-Za-z0-9_-]{10,}\.eyJ[A-Za-z0-9_-]{10,}\.[A-Za-z0-9_-]+` (weight 0.8)
   - `private_key_block`: `-----BEGIN\s+(RSA|EC|OPENSSH|PGP)\s+PRIVATE\s+KEY-----` (weight 0.95)
   - `connection_string`: `(?i)(postgres|mysql|mongodb|redis)://[^\s]+:[^\s]+@` (weight 0.9)
   - `password_in_output`: `(?i)(password|passwd|pwd)\s*[:=]\s*\S{4,}` (weight 0.7)

   **`encoding_evasion`** (encoded payloads that decode to injection):
   - `base64_instruction_block`: detect base64-encoded strings > 20 chars that decode to ASCII text containing injection keywords (weight 0.85) -- this is checked in the decoded pass, not raw
   - `hex_encoded_payload`: `(?i)(\\x[0-9a-f]{2}){8,}` (weight 0.7)
   - `unicode_escape_sequence`: `(\\u[0-9a-fA-F]{4}){4,}` (weight 0.7)
   - `html_entity_chain`: `(&#x?[0-9a-fA-F]+;){4,}` (weight 0.7)
   - `rot13_marker`: `(?i)(rot13|ebg13|decode\s+this)` (weight 0.6)

   **`exfil_directive`** (instructions in tool output to exfiltrate data):
   - Adapt PRH data_exfil patterns: `exfil_via_url`, `extract_env_vars`
   - `send_data_instruction`: `(?i)(send|post|transmit|upload|exfiltrate?)\s+(this|the|all)\s+(data|content|output|response)\s+to\s+` (weight 0.85)
   - `curl_exfil`: `(?i)curl\s+(-X\s+POST\s+)?https?://[^\s]+` in tool output context (weight 0.7)

3. Export `SANITIZER_CATEGORIES` dict mapping category name to list of PatternDef.

4. Also export `ALL_SANITIZER_PATTERNS` -- flat list of all patterns for quick iteration.

### Step 2: Create multi-pass decoder
**File:** `src/aletheia/sanitizer_decoder.py`

Implements the multi-pass decoding pipeline that produces multiple representations of the input for pattern matching.

1. `decode_passes(raw: bytes, max_bytes: int = 1_048_576) -> list[tuple[str, str]]`:
   Returns list of `(pass_name, decoded_text)` tuples:

   - **Pass 1 "raw"**: `raw.decode("utf-8", errors="replace")` -- truncate at max_bytes
   - **Pass 2 "base64"**: Scan for base64-looking strings (regex: `[A-Za-z0-9+/=]{20,}`), attempt `base64.b64decode()` on each, concatenate successfully decoded fragments. Only if any fragment decodes to valid UTF-8 text.
   - **Pass 3 "unicode_normalized"**: Apply `unicodedata.normalize("NFKC", raw_text)` -- collapses homoglyphs, fullwidth chars, compatibility forms
   - **Pass 4 "html_decoded"**: Apply `html.unescape()` to raw_text -- decodes `&#x41;` -> `A`, `&lt;` -> `<`, etc.

   Each pass is independent (all derived from raw input, not chained). This ensures a single evasion technique does not compound.

2. Performance: each pass must complete in < 10ms for 1MB input. Base64 pass uses regex to find candidates first (no brute-force decode of entire output).

### Step 3: Create SanitizerEngine
**File:** `src/aletheia/sanitizer_engine.py`

Core scanning engine that orchestrates multi-pass decoding and pattern matching.

1. `SanitizerResult` dataclass:
   - verdict: str ("clean"|"warn"|"block")
   - patterns_matched: list[dict] -- each with pattern_id, category, severity (from weight: >=0.9 high, >=0.7 medium, else low), match_location (start, end), snippet_hash (sha512 of matched text)
   - scan_duration_ms: float
   - passes_run: list[str]
   - partial_scan: bool (True if input exceeded 1MB)

2. `SanitizerEngine` class:
   - `__init__(self, patterns: dict[str, list[PatternDef]] = None)` -- defaults to SANITIZER_CATEGORIES from sanitizer_patterns.py
   - `scan(self, output: bytes, mode: str = "warn") -> SanitizerResult`:
     a. If mode == "passthrough", return clean result immediately (0ms scan)
     b. Start perf timer
     c. Call `decode_passes(output)` to get all text representations
     d. For each pass, for each category, run pattern matching (same probabilistic scoring as PRHAnalyzer._score_category)
     e. Collect all unique pattern matches (deduplicate across passes by pattern_id)
     f. Determine verdict:
        - No matches -> "clean"
        - Any match with weight >= 0.9 -> "block" (if mode is "block"), else "warn"
        - Any match with weight >= 0.5 -> "warn"
        - Below 0.5 -> "clean"
     g. If mode == "warn", cap verdict at "warn" (never "block")
     h. If mode == "block", use actual verdict
     i. Record scan_duration_ms, return result

   - `_match_patterns(self, text: str, pass_name: str) -> list[dict]`:
     For each category, for each pattern, search text. Return list of match dicts.

3. Module-level singleton (same pattern as tool_policy_engine):
   - `_active_sanitizer: Optional[SanitizerEngine] = None`
   - `init_sanitizer() -> None`
   - `get_active_sanitizer() -> Optional[SanitizerEngine]`

### Step 4: Create POST /v1/aletheia/sanitize endpoint
**File:** `src/aletheia/sanitize_router.py`

1. FastAPI APIRouter with prefix `/v1/aletheia`, tags `["Aletheia Sanitizer"]`

2. Pydantic request model `SanitizeRequest`:
   - tool: str
   - command: str
   - output: str (the raw tool stdout as string -- tiresias-exec sends it)
   - agent_id: str
   - tenant_id: str
   - mode: str = "warn" (passthrough|warn|block)

3. Pydantic response model `SanitizeResponse`:
   - verdict: str
   - patterns_matched: list[dict]
   - scan_duration_ms: float
   - sanitized_output: Optional[str] = None (only set when mode=block and verdict=block)

4. `POST /sanitize` endpoint:
   - Get engine from `get_active_sanitizer()`
   - If no engine, return `{"verdict": "clean", "patterns_matched": [], "scan_duration_ms": 0}`
   - Convert request.output to bytes (UTF-8)
   - Call `engine.scan(output_bytes, mode=request.mode)`
   - If verdict == "block":
     - Set `sanitized_output = "[BLOCKED: prompt injection detected in tool response]"`
     - Log blocked event with structlog (include agent_id, command, pattern summary)
     - Store original output encrypted for forensics: call a helper that writes to DB (aletheia_blocked_outputs table) or reuse existing AES-256-GCM envelope from `src/aletheia/encryption.py`
   - Return SanitizeResponse

5. Wire into `src/main.py`

```go
result := executeCommand(command)

// Sanitize output if mode is not passthrough
sanitizerResult := &SanitizeResponse{Verdict: "skipped"}
if identity.SanitizeMode != "passthrough" && !flags.forceOffline && identity.SoulWatchURL != "" {
    resp, err := sanitizeOutput(identity, command, result.Stdout)
    if err == nil && resp != nil {
        sanitizerResult = resp
        if resp.Verdict == "block" && resp.SanitizedOutput != nil {
            // Replace stdout with sanitized message
            fmt.Fprintf(os.Stderr, "tiresias-exec: OUTPUT BLOCKED (%d patterns matched)\n", len(resp.PatternsMatched))
            os.Stdout.Write([]byte(*resp.SanitizedOutput))
            os.Stderr.Write(result.Stderr)
        } else {
            if resp.Verdict == "warn" {
                fmt.Fprintf(os.Stderr, "tiresias-exec: WARNING: %d suspicious patterns in output\n", len(resp.PatternsMatched))
            }
            os.Stdout.Write(result.Stdout)
            os.Stderr.Write(result.Stderr)
        }
    } else {
        // Fail-open: pass output through
        os.Stdout.Write(result.Stdout)
        os.Stderr.Write(result.Stderr)
    }
} else {
    os.Stdout.Write(result.Stdout)
    os.Stderr.Write(result.Stderr)
}
```
