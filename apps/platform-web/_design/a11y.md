# Pantheon Accessibility Contract

**Status:** authoritative (Wave E.4 — UI Depth Wave coherence pass).
**Companion docs:** `_design/tokens.md`, `_design/visual-language.md`.
**Source of truth in code:** `apps/platform-web/eslint.config.mjs` (lint), `src/__tests__/a11y/*` (axe).

This document is the contract every Pantheon UI primitive and surface MUST satisfy. If a primitive shipped by `_shared/` cannot meet a clause here, the primitive is wrong — not the contract.

---

## 1. Target — WCAG 2.2 Level AAA

We target **WCAG 2.2 Level AAA** for the following surfaces:

- **Primary text** (`text-text-primary` / `#f1f3f9`) on any surface in the ladder — passes AAA Normal (7:1) on every surface step.
- **Secondary text** (`text-text-secondary` / `#a4acbf`) on any surface — passes AAA Normal (7:1) on `surface-0` / `surface-1` / `surface-2` / `surface-3`.
- **UI components + interactive contrast** (focus rings, borders, accent button surfaces) at AAA UI Components (3:1) — universally met today by `--accent-base` on the surface ladder, `border-strong` on `surface-2`, and per-OS accent badges on `surface-2`.

### Documented carve-outs

| Token / surface           | Level we hit | Why we don't push to AAA Normal |
|---------------------------|--------------|---------------------------------|
| `text-text-tertiary` (`#8b93a7`) on `surface-2` | **AA Normal (5.2:1)** + AAA Large (4.5:1) | Tertiary text reads as eyebrows, meta labels, helper copy — pushing it to AAA Normal would collapse the visible distinction from `text-text-secondary`. The 3-tier hierarchy is the legibility contract; flattening it would hurt scannability more than the contrast win. We document this as the single carve-out and never use `text-text-tertiary` for primary or secondary body copy. |
| Per-OS accent badges (e.g. `bg-os-cyber/15 text-os-cyber` on `surface-2`) | **AAA UI Components (3:1)** | Treated as incidental UI / identifying tints, not body text. Per spec we audit them at the UI Components threshold, which they meet universally. |

These carve-outs are the **only** exceptions. Any new token or surface that doesn't fit must be explicitly added to this table — silently introducing a non-AAA primary or secondary text path is a bug.

## 2. ARIA patterns we adopt

For each interaction type below, Pantheon adopts the WAI-ARIA Authoring Practices Guide (APG) recipe. Primitives in `_shared/views/` and `_shared/*.tsx` are the canonical implementations; consumers should reuse them rather than re-rolling ARIA per surface.

| Pattern              | Recipe                                                                                     | Pantheon primitive                                  |
|----------------------|--------------------------------------------------------------------------------------------|-----------------------------------------------------|
| **Buttons**          | Native `<button type="button">`. Never `<div onClick>`.                                    | All `_shared/*` use `<button>`.                     |
| **Links**            | Native `<a href>` (or Next `<Link>`). `aria-current="page"` for active nav.                 | `Sidebar`, `MobileNav`, `WizardForm` step links.    |
| **Forms**            | Native `<label for>` / `<label><input/></label>`. `aria-describedby` on error / helper.    | All `_shared/` forms; `WizardForm` step nav.        |
| **Dialogs**          | `role="dialog" aria-modal="true" aria-labelledby="<title-id>"`. Focus-trap via `inert` on background, `Escape` closes, focus restored on close. | `MobileNav` drawer. |
| **Tabs**             | `role="tablist"` + `role="tab"` w/ `aria-selected`. Roving `tabindex` (ArrowLeft/Right). `aria-controls` → `role="tabpanel"`. | `CrossEntityTabs` (uses role-tab semantics where applicable). |
| **Listboxes**        | `role="listbox"` + `role="option"` w/ `aria-selected`. Single-select default.              | `EntitySearch` results dropdown, `Combobox`.        |
| **Combobox**         | APG combobox: wrapper `role="combobox" aria-expanded aria-controls aria-haspopup aria-autocomplete`. Input drives `aria-activedescendant` over options in the listbox. | `Combobox`. |
| **Live regions**     | `role="status"` for polite updates (loading, saved confirmations), `role="alert"` for assertive errors. `aria-busy="true"` on the loading wrapper. | `Skeleton` / `SkeletonGroup`, dnd-kit `announcements`. |
| **Focus management** | Focus restoration on dialog close. Initial focus to first interactive on dialog open. Visible focus via `*:focus-visible { outline: 2px solid var(--accent-base); outline-offset: 2px; }` in `globals.css`. | `MobileNav`, all primitives via globals. |
| **Keyboard interaction** | Every interactive primitive supports keyboard equivalents: ArrowUp/Down/Enter/Esc on listboxes + combobox; ArrowLeft/Right + Home/End on tabs; Esc closes dialogs. | `Combobox`, `EntitySearch`, `CrossEntityTabs`, `MobileNav`, `KanbanBoard` (dnd-kit `KeyboardSensor`). |

## 3. Reduced motion

We honor `@media (prefers-reduced-motion: reduce)` as a hard contract. The global rule in `globals.css` enforces:

- All `animation-duration` collapses to `0.01ms`, `animation-iteration-count` clamps to `1`, `transition-duration` collapses to `0.01ms`, `scroll-behavior` falls back to `auto`.
- `animate-pulse` (used by `Skeleton`) is fully disabled (`animation: none`) — the shimmer is the one motion users with motion-sensitivity feel most acutely, so we hard-stop it rather than ramp it.
- View transitions (`@view-transition { navigation: auto; }`) are implicitly disabled because the cross-document transition uses `transition-duration`, which collapses to instant under the rule.

Primitives MUST NOT roll their own motion outside `transition` / `transition-slow` (the tokens documented in `tokens.md` §9). Anything that does is automatically out of contract with reduced-motion.

## 4. ESLint enforcement

The platform-web ESLint config enables **`eslint-plugin-jsx-a11y`'s recommended preset at error level**. The full rule list is owned by the plugin — we deliberately do not pin it here so plugin updates land automatically.

Local violations: if the recommended preset surfaces a violation in a file Wave E.4 cannot reasonably fix in-scope (typically because the fix is owned by a different sub or upstream wave), we add a single targeted `// eslint-disable-next-line jsx-a11y/<rule> -- <reason>` comment at the violation site. No global rule downgrades; no `warn` levels — enforcement is theatrical otherwise.

Carve-out exception list (kept short by design): consult `git log -p` for the inline disable comments. If the list grows past ~5 sites, that's a signal a separate fix-it sub is needed.

## 5. Testing

Two layers:

### 5.1 Vitest + vitest-axe (primitives)

Every primitive in `_shared/` ships an axe pass via `vitest-axe`'s `toHaveNoViolations` matcher (registered in `src/__tests__/setup.ts`). The suite lives at `src/__tests__/a11y/primitives.test.tsx`. One `it('renders without a11y violations')` per primitive, exercised with minimal but representative props.

Coverage today:
- `Skeleton`, `Spinner` (incl. with `label`)
- `EmptyState` (with primary + secondary CTAs)
- `DashboardWidget` (with labelled body)
- `ActivityFeed`
- `KindFilterChips`
- `SavedViews`
- `BulkActionsBar`
- `CrossEntityTabs`
- `EntitySearch` (incl. with filter / sort / view-toggle)
- `KanbanBoard`
- `Combobox`
- `WizardForm`
- `CoachNotConfigured`

### 5.2 Surface-level (deferred)

We deliberately do **not** ship the `@axe-core/react` dev runtime injection — it adds runtime overhead, and the violations it would catch are already gated by jsx-a11y at lint and vitest-axe at the primitive layer. Surface-level (page-route) axe sweeps are a future wave's call; until then, primitives are the contract surface.

---

## 6. Maintenance

- Adding a new primitive: add an axe test in `primitives.test.tsx` AND verify the new file passes lint with no inline disables.
- Adding a new token: re-check the contrast table (§1). Anything < AAA Normal for primary / secondary text is a bug.
- Bumping `eslint-plugin-jsx-a11y`: read the changelog. If a new rule lands as `error` in the recommended preset, run lint locally and fix-in-scope or document the disable.

*End of contract. Wave E.4 ships this; subsequent waves consume.*
